## 架构变更

### 1. 新增文件

#### `src/messaging/markdown-filter.ts` (362行)
**全新的流式Markdown过滤器** - 字符级状态机，实时剥离不支持的Markdown语法。

**设计特点**:
- **流式处理**: `feed()` + `flush()` 模式，支持增量文本处理
- **最小缓冲**: 仅保留模式消歧所需的字符（如可能变成`***`的尾随`*`）
- **状态机**: `sol` (行首) → `body` (行体) → `fence` (代码块) → `inline` (行内标记)

**保留的语法**:
- 代码围栏 (```)
- 行内代码 (`)
- 表格 (`|...|`)
- 水平线 (`---`, `***`, `___`)
- 粗体 (`**`/`***`)
- 非CJK内容的斜体/粗斜体

**过滤的语法** (标记剥离，内容保留):
- CJK内容的斜体/粗斜体包装
- H5/H6标题 (`#####`, `######`)
- 图片 (`![alt](url)`) - 完全移除

**CJK检测**:
```typescript
/[\u2E80-\u9FFF\uAC00-\uD7AF\uF900-\uFAFF]/.test(text)
```

### 2. 删除文件

#### `src/weixin-cli.ts`
- **移除CLI子命令**，统一使用 `openclaw plugins uninstall` 流程
- 简化插件安装流程

## 核心模块变更

### 1. API层 (`src/api/api.ts`)

**变更: 超时处理变为可选**

```diff
- timeoutMs: number
+ timeoutMs?: number
```

**影响**: 移除强制超时，允许服务器端或堆栈限制控制超时

**实现细节**:
- `timeoutMs` 为 `undefined` 或 `<= 0` 时，不创建 `AbortController`
- 仅在明确指定超时时才设置 `setTimeout`

### 2. 认证登录 (`src/auth/login-qr.ts`)

**变更: 移除QR码获取的客户端超时**

```diff
- const GET_QRCODE_TIMEOUT_MS = 5_000;
- timeoutMs: GET_QRCODE_TIMEOUT_MS,
+ // 移除固定超时，依赖服务器/堆栈限制
```

**原因**: 允许QR码请求根据网络条件自适应超时

### 3. 消息发送 (`src/messaging/send.ts`)

**重大变更: Markdown处理重构**

#### v2.1.1 (旧)
```typescript
import { stripMarkdown } from "openclaw/plugin-sdk/text-runtime";

export function markdownToPlainText(text: string): string {
  // 基于正则表达式的批量处理
  result = result.replace(/```[^\n]*\n?([\s\S]*?)```/g, ...);
  result = stripMarkdown(result);
  return result;
}
```

#### v2.1.8 (新)
```typescript
export { StreamingMarkdownFilter } from "./markdown-filter.js";

// 在 channel.ts 和 process-message.ts 中使用:
const f = new StreamingMarkdownFilter();
const filteredText = f.feed(rawText) + f.flush();
```

**改进点**:
1. **从批量处理 → 流式处理**: 支持大文本增量处理
2. **更精细的控制**: 保留非CJK内容的格式，只剥离CJK格式
3. **性能提升**: 字符级状态机比多次正则替换更高效

**删除的导出**:
- ~~`markdownToPlainText()`~~

### 4. 通道插件 (`src/channel.ts`)

**变更1: 工具调用路径应用Markdown过滤**

```typescript
// 新增 StreamingMarkdownFilter 导入
import { sendMessageWeixin, StreamingMarkdownFilter } from "./messaging/send.js";

async function sendWeixinOutbound(params: {...}) {
  // ...
  const f = new StreamingMarkdownFilter();
  const rawText = params.text ?? "";
  const filteredText = f.feed(rawText) + f.flush();  // 新增过滤
  const result = await sendMessageWeixin({ to: params.to, text: filteredText, ... });
}
```

**影响**: tool-call 输出路径现在与 model-output 路径一致，都应用流式过滤

**变更2: 延迟导入监控模块**


## 配置变更

### 登录后配置重载 (v2.1.2)

**行为变更**:
- 每次成功登录后，更新 `openclaw.json` 中的 `channels.openclaw-weixin.channelConfigUpdatedAt` (ISO 8601)
- 网关会检测此时间戳并重新加载磁盘配置
- **替代方案**: 不再写入空的 `accounts: {}` 占位符

## Bug修复

### 1. 插件注册重入 (v2.1.7)

**问题**: 插件注册时拉入 `monitor` → `process-message` → `command-auth` 链，可能在账户启动前重新进入插件/提供者注册表

**解决方案**:
1. 延迟导入 `monitorWeixinProvider` 到 `startAccount` 内部
2. 延迟导入 `resolveSenderCommandAuthorizationWithRuntime` / `resolveDirectDmAuthorizationOutcome` 到运行时

### 2. 初始化副作用 (v2.1.7)

**问题**: 模块初始化时触发 `ensureContextWindowCacheLoaded`，导致 `loadOpenClawPlugins` 重入

**解决方案**: 在 `process-message.ts` 中延迟导入命令鉴权函数

### 3. OpenClaw 2026.3.31+ 兼容性 (v2.1.2)

**问题**: 安装插件时触发 "dangerous code pattern" 警告

**解决**: 移除 `weixin-cli.ts` 中的CLI子命令注册

## 升级建议

### 对现有部署的影响

1. **配置重载行为变更**: 登录后会更新配置文件时间戳
2. **Markdown渲染变更**: 非CJK内容的格式会被保留（如英文斜体）
3. **超时行为**: QR码获取和网络请求可能等待更长时间

### 兼容性

- **向后兼容**: v2.1.1 → v2.1.8 无破坏性变更
- **OpenClaw版本**: 推荐使用 OpenClaw 2026.3.31+ 以避免警告

3. **CJK感知**: 区分CJK和非CJK内容的格式处理
